Práctica: Agente con opencode + Ollama (local)

Configuración, instalación y ejecución de un agente usando un modelo local (Qwen2.5 Coder 7B).

Captura 1

Configuración de opencode para usar Ollama (local)

En esta captura se edita el archivo ~/.config/opencode/config.json para que opencode apunte a un proveedor local.

  • Se define el provider como ollama con interfaz compatible tipo OpenAI.
  • Se fija el baseURL a http://127.0.0.1:11434/v1, que es el endpoint local de Ollama.
  • En models se registra el modelo (por ejemplo qwen2.5-coder:7b) y se selecciona en model para que sea el usado por defecto.
Captura 2

Instalación y preparación del directorio de configuración

Esta captura muestra los comandos necesarios para dejar opencode instalado y listo para configurar.

  • Se instala globalmente con sudo npm install -g opencode-ai --force (aparece el aviso de que --force desactiva protecciones recomendadas).
  • Se crea el directorio de configuración con mkdir -p ~/.config/opencode.
  • Se abre el fichero de configuración con nano ~/.config/opencode/config.json para editar los parámetros del proveedor/modelo.
Captura 3

Ejecución del agente en opencode (modelo local)

Aquí se ve la interfaz de opencode ejecutando una petición en español (por ejemplo: “cuéntame un cuento”).

  • El agente muestra la llamada/uso de una habilidad (por ejemplo storytelling) para orientar la respuesta.
  • Se aprecia el modelo seleccionado (qwen2.5-coder:7b) funcionando en local (Ollama).
  • Se muestra un monitor de uso de GPU (“Apple M4 (Integrada)”), indicando que el equipo está cargando trabajo durante la ejecución.
Captura 4

Lanzamiento de opencode desde el proyecto

En esta captura se entra al directorio del proyecto y se ejecuta opencode desde terminal.

  • Se cambia al proyecto con cd /Users/.../Desktop/AgenteIA.
  • Se lanza el entorno del agente con el comando opencode, que usa la configuración creada en ~/.config/opencode/config.json.
