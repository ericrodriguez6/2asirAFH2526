Memoria de Fine-tuning
Gemma 3‑270M‑it con MLX‑LM‑LoRA

Secuencia de pasos desde la creación del entorno hasta el entrenamiento exitoso del modelo.

Paso 1 – Creación del entorno e instalación de dependencias

En este primer paso creo el directorio gemma-finetune-v2, genero y activo el entorno virtual de Python y actualizo pip. A continuación instalo los paquetes necesarios (mlx-lm, datasets, huggingface_hub, accelerate) para poder cargar y afinar el modelo Gemma en Apple Silicon.

Instalación del entorno virtual y de las librerías necesarias para MLX‑LM.

Paso 2 – Autenticación en Hugging Face

Configuro el acceso a Hugging Face ejecutando huggingface-cli login. Introduzco el token personal para poder descargar el modelo base Gemma y, si lo necesito, subir después los pesos afinados al Hub.

Proceso de login en Hugging Face desde la terminal.

Paso 3 – Primeros ficheros de datos en JSONL

Creo el directorio data/ y genero train.jsonl y valid.jsonl con los primeros ejemplos de conversación en formato chat. Uso head para comprobar el contenido, aunque en este intento aparece un error al interpretar el comentario # Verifica como nombre de archivo.

Generación manual de los primeros ficheros train.jsonl y valid.jsonl.

Paso 4 – Intento de lanzar el comando de LoRA

Intento iniciar el entrenamiento ejecutando mlx_lm.lora con el modelo google/gemma-3-270m-it y los parámetros de entrenamiento. El shell devuelve command not found porque el ejecutable correcto del paquete mlx-lm-lora es mlx_lm_lora.train.

Primer intento fallido de lanzar el entrenamiento con el comando incorrecto.

Paso 5 – Creación de un archivo de configuración YAML

Para simplificar los parámetros preparo un fichero train.yaml donde defino el modelo, la ruta de datos, el número de iteraciones, el batch_size y la ruta de salida de los adaptadores LoRA. Al ejecutar mlx_lm.lora --config train.yaml el sistema sigue sin encontrar el comando, lo que confirma que debo usar la interfaz mlx_lm_lora.train.

Definición de la configuración de entrenamiento en formato YAML.

Paso 6 – Primer entrenamiento: error por tamaño de dataset

Una vez identificado el ejecutable correcto, lanzo el entrenamiento con mlx_lm_lora.train y se empieza a cargar el modelo Gemma. Durante la fase de evaluación aparece el mensaje ValueError: Dataset must have at least batch_size*2 examples but only has 1, indicando que el conjunto de validación es demasiado pequeño para el batch_size elegido.

El entrenamiento se detiene porque el conjunto de validación sólo tiene un ejemplo.

Paso 7 – Regeneración de los ficheros de entrenamiento y validación

Para solucionar el problema elimino el directorio data, lo vuelvo a crear y genero de nuevo train.jsonl con varios mensajes de usuario y respuestas del asistente centrados en ASIR, Apache y fine‑tuning. También creo un valid.jsonl con más ejemplos y reviso el contenido con cat para asegurar que cada línea es un JSON válido.

Nueva generación de los ficheros de datos con más ejemplos y formato correcto.

Paso 8 – Carga del dataset: error de índice fuera de rango

Al relanzar el entrenamiento, el script descarga los ficheros del modelo pero falla al cargar el dataset local con el error IndexError: list index out of range. Esto indica que la lista de ejemplos está vacía o el formato de las líneas no se está interpretando correctamente por el cargador de datos.

Fallo al crear el dataset a partir de los ficheros JSONL locales.

Paso 9 – Ajuste del contenido de los JSONL y nuevo intento

Vuelvo a editar train.jsonl y valid.jsonl, manteniendo un ejemplo por línea y el formato de mensajes requerido por mlx-lm-lora. Relanzo el entrenamiento con los mismos parámetros; aunque se avanza más, todavía debo ajustar el tamaño del conjunto de validación.

Relanzando el entrenamiento tras corregir la estructura de los ficheros de datos.

Paso 10 – Ampliación definitiva de train y valid

Amplío el número de ejemplos en train.jsonl y valid.jsonl, añadiendo nuevas instrucciones sobre bases de datos relacionales y comandos básicos de Linux. Así garantizo que el conjunto de validación cumple la condición de tener al menos batch_size*2 ejemplos para que el evaluador funcione sin errores.

Ampliación del dataset con más ejemplos para evitar errores en la evaluación.

Paso 11 – Entrenamiento exitoso del modelo Gemma afinado

Con los datasets ya corregidos, ejecuto de nuevo mlx_lm_lora.train sobre el modelo google/gemma-3-270m-it, durante 50 iteraciones y batch_size 2. El log muestra la evolución de la pérdida de entrenamiento y validación y, al finalizar, se guardan los pesos LoRA en my-gemma-lora/adapters.safetensors, dejando listo el modelo afinado para su uso posterior.

Entrenamiento completado y guardado de los adaptadores LoRA del modelo Gemma.

Paso 12 – Modelos Gemma 3‑270M‑it en formato MLX

Como referencia de la arquitectura, consulto la lista de variantes de gemma-3-270m-it disponibles en formato MLX (bf16, 4‑bit, 8‑bit, etc.) en el repositorio de la comunidad. Esta información me permite elegir la versión adecuada del modelo base para el fine‑tuning local.

Variantes del modelo Gemma 3‑270M‑it listas para usar con MLX‑LM.
